Bilan des compétences SAE

SAE (S3)

Contexte de la SAE :
Dans le cadre de cette SAÉ, notre mission principale consistait à concevoir et développer une application web complète, en collaborant au sein d'une équipe de cinq étudiants. L'enjeu de ce projet était de répondre à la demande spécifique d'un client (représenté par l'un de nos professeurs) nécessitant un système de diffusion sonore centralisé. Concrètement, nous devions imaginer et livrer une plateforme permettant à un administrateur de gérer, d'organiser et de déclencher des flux audio sur différents lecteurs distants, le tout via une interface web intuitive et réactive.

Les tâches que j'ai effectuées:
Pour mener à bien ce projet, nous avons structuré notre approche en nous répartissant les missions de la manière suivante :

  • Recueil du besoin : Élaboration des user stories, rédaction du cahier des charges et échanges avec le client pour identifier ses besoins précis. (Travail en équipe)
  • Gestion de projet : Mise en place d'une feuille de route pour la planification et la répartition des tâches. (Travail en équipe)
  • Développement web : Utilisation du framework Flask (Python), HTML, CSS et JavaScript pour la gestion des requêtes HTTP. (Tâche individuelle)
  • Logique métier : Création des outils de gestion des fichiers MP3 et développement de la page de supervision des lecteurs. (Tâche individuelle)
  • Gestion des données : Conception et mise en place d'une base de données avec SQLite. (Tâche individuelle)

Bilan des compétences acquises :
Cette première expérience de projet à grande échelle m'a permis de consolider plusieurs acquis essentiels pour mon futur métier :

  • Travail en équipe et gestion de projet : Capacité à collaborer efficacement au sein d'un groupe, notamment par la structuration des tâches via une feuille de route et une répartition claire des responsabilités.
    Livrable associé : Consulter notre fiche de route (PDF)
  • Développement Back-end : Maîtrise de la logique métier côté serveur et gestion des requêtes HTTP avec le framework Flask.
    Livrable associé : Voir le code source sur GitHub
  • Gestion et manipulation de données : Conception, implémentation et requêtage de bases de données relationnelles avec SQLite.
    Livrable associé : Lire le compte rendu de la base de données (PDF)

Pistes d'amélioration :
Avec le recul, certains aspects de notre organisation auraient pu être optimisés :

  • Mettre en place des tests plus rigoureux : Face aux contraintes de temps, nous n'avons pas pu tester notre application de manière aussi exhaustive que nous l'aurions souhaité pour garantir une fiabilité totale.
  • Améliorer la documentation : Actuellement réduite au strict minimum, la documentation de notre projet nécessiterait un effort de compréhension important si nous devions nous replonger dans ce code plusieurs mois plus tard. C'est un point de vigilance pour la maintenabilité de mes futurs projets.

SAE (S4)

Contexte de la SAE :
Pour cette deuxième phase, l'objectif s'inscrivait dans une véritable démarche de maintenance et d'évolution logicielle. Au lieu de démarrer un projet de zéro, le défi consistait à reprendre le projet de diffusion sonore initié lors du semestre précédent (qu'il s'agisse du nôtre ou de celui d'un autre groupe). Notre mission était d'auditer le code existant, d'en comprendre l'architecture technique, puis d'y apporter les corrections et améliorations nécessaires pour aboutir à une version robuste, finalisée et pleinement fonctionnelle.

Les tâches que j'ai effectuées:
Travailler sur un code déjà existant nous a obligés à adapter notre méthode de travail :

  • Évaluation et sélection de projets : Mise en place de critères d'évaluation du code, notation des différents projets du S3 et sélection de celui ayant obtenu la meilleure note pour l'améliorer. (Travail en équipe)
  • Refonte de la base de données : Amélioration et optimisation de la structure de la base de données existante. (Travail en binôme - réalisé pendant que le reste de l'équipe évaluait les projets)
  • Développement Back-end : Correction et finalisation de la fonctionnalité de gestion des fichiers MP3. (Tâche individuelle)
  • Débogage Front-end : Réparation du calendrier de diffusion des musiques en fonction des jours de la semaine, une fonctionnalité qui était cassée sur le projet initial. (Tâche individuelle)

Bilan des compétences acquises :
Cette SAÉ s'est révélée particulièrement formatrice, notamment sur ma capacité d'adaptation :

  • Lecture et appropriation de code existant : C'est pour moi la compétence la plus importante acquise ici. Être capable de lire et de comprendre un ancien code (le sien ou celui d'un autre) est crucial, car dans mon avenir professionnel, je serai constamment amené à lire le code d'autres développeurs ou à corriger un bug sur un de mes anciens projets.
  • Maîtrise des bonnes pratiques GitHub : Lors de la SAÉ du S3, nous utilisions GitHub d'une manière qui était souvent erronée. Grâce à l'expérience acquise en stage, j'ai appris à l'utiliser de la bonne façon, et j'ai pu prendre le temps d'apprendre ces bonnes pratiques aux membres de mon équipe qui n'étaient pas encore à l'aise avec l'outil.
    Livrable associé : Voir le code source de notre projet S4 sur GitHub
  • Consolidation des acquis techniques : Réinvestissement de toutes les compétences de développement et de gestion mobilisées lors de la SAÉ du S3, mais avec un niveau d'exigence et de maturité supérieur.

Pistes d'amélioration :
Tout comme pour le semestre précédent, nous devons encore progresser sur la rigueur de livraison :

  • Conception de tests et documentation : Les axes d'amélioration restent similaires à ceux du S3. Par manque de temps, nous n'avons pas pu concevoir des tests suffisamment rigoureux ni rédiger une documentation à la hauteur de la qualité de notre projet. Je devrai mieux gérer cet aspect à l'avenir.
